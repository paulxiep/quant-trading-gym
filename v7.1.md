# V7.1: Gym Environment + PyO3 Bindings - Implementation Plan

## Context

V7.1 delivers complete RL infrastructure: Rust gym environment (`crates/gym/`) and Python bindings (`crates/gym-python/`). This enables V7.2 RL training with stable-baselines3 or custom Python training code.

**Scope**: Two new crates providing gymnasium-compatible environment + PyO3 Python bindings
**Requires**: V6.3 complete (`CanonicalFeatures` extractor, 28 SHAP-validated features)
**Estimated**: ~1 week (gym: 3-4 days, PyO3: 2-3 days, testing: 2 days)

**Key architectural decisions:**
1. **Multi-symbol from start**: Portfolio-level observations (N × 28 features), per-symbol MultiDiscrete actions
2. **Clean SoC**: External agent stored separately in `AgentOrchestrator.external_agent`, no Arc<Mutex>
3. **Correct reward**: Explicit `prev_equity: f64` field tracking for accurate P&L delta
4. **Zero parallelism impact**: Action mutation happens before `sim.step()`, not during Phase 1
5. **GIL release**: All Rust computation runs inside `py.allow_threads()` for parallel environments

---

## Overview

### Part 1: Gym Environment (Rust)

New crate `crates/gym/` providing a gymnasium-compatible `TradingEnv` that wraps a full `Simulation` (with background agents) and exposes a single RL agent slot via step/reset/seed API.

**Key features:**
- **Multi-symbol support**: Portfolio-level observations and actions (N symbols × 28 features)
- **Clean SoC**: External agent stored separately in AgentOrchestrator (no downcasting)
- **Correct reward computation**: Explicit `prev_equity` tracking for accurate P&L delta
- **Deterministic episodes**: Seed support for reproducible training

### Part 2: PyO3 Bindings (Python ↔ Rust)

New crate `crates/gym-python/` providing Python bindings via PyO3. Enables Python training code (scikit-learn, stable-baselines3) to call the Rust gym environment.

**Key features:**
- **GIL release**: All Rust computation runs inside `py.allow_threads()` for parallelism
- **NumPy arrays**: Zero-copy where possible, gymnasium-compatible interface
- **Vectorized environments**: `PyVecTradingEnv` runs N simulations in parallel via rayon
- **gymnasium compatibility**: `observation_space` and `action_space` properties return actual `gymnasium.spaces` objects

**Direction**: Python calls Rust (training), NOT Rust calls Python (inference). Inference remains pure Rust (V6 ensemble, future V7.2 RL agent).

## Design Decisions

### 1. One Simulation Per Episode (Unchanged)

`TradingEnv::reset()` creates a **fresh `Simulation`** rather than resetting an existing one. Rationale:
- Avoids complex subsystem reset logic (14 phases, 6 subsystems)
- `Simulation::new()` is fast (no heavy initialization)
- Guarantees clean state — no subtle bugs from incomplete resets
- Seed support is natural: pass seed to `SimulationConfig`

### 2. ExternalAgent with Direct Mutation (MODIFIED - FINAL DESIGN)

**Original problem**: Discrete actions hardcoded, V7 continuous actions require modifying ExternalAgent

**Finalized design**: Multi-symbol ExternalAgent with clean SoC (no Arc<Mutex>)

```rust
/// V6 discrete action enum
#[derive(Debug, Clone, Copy, PartialEq)]
pub enum DiscreteAction {
    Sell = 0,
    Hold = 1,
    Buy = 2,
}

impl DiscreteAction {
    pub fn from_idx(idx: usize) -> Self {
        match idx {
            0 => DiscreteAction::Sell,
            1 => DiscreteAction::Hold,
            2 => DiscreteAction::Buy,
            _ => panic!("Invalid action index: {}", idx),
        }
    }
}

/// External agent for gym control (multi-symbol support)
pub struct ExternalAgent {
    id: AgentId,
    state: AgentState,
    next_actions: HashMap<Symbol, DiscreteAction>,  // Per-symbol actions
    order_size: Quantity,
    config: ExternalAgentConfig,
}

impl ExternalAgent {
    pub fn new(id: AgentId, config: ExternalAgentConfig) -> Self {
        Self {
            id,
            state: AgentState::new(config.initial_cash),
            next_actions: HashMap::new(),
            order_size: config.order_size,
            config,
        }
    }

    /// Set next actions for all symbols (called by TradingEnv before tick)
    pub fn set_next_actions(&mut self, actions: HashMap<Symbol, DiscreteAction>) {
        self.next_actions = actions;
    }

    fn generate_buy_order(&self, ctx: &StrategyContext, symbol: &Symbol) -> Vec<Order> {
        // Check position limits, cash availability, etc.
        vec![Order::market_buy(symbol.clone(), self.order_size, self.id)]
    }

    fn generate_sell_order(&self, ctx: &StrategyContext, symbol: &Symbol) -> Vec<Order> {
        // Check position exists, etc.
        vec![Order::market_sell(symbol.clone(), self.order_size, self.id)]
    }
}

impl Agent for ExternalAgent {
    fn id(&self) -> AgentId { self.id }

    fn on_tick(&mut self, ctx: &StrategyContext<'_>) -> AgentAction {
        let mut orders = vec![];

        for (symbol, action) in &self.next_actions {
            match action {
                DiscreteAction::Sell => orders.extend(self.generate_sell_order(ctx, symbol)),
                DiscreteAction::Hold => {},
                DiscreteAction::Buy => orders.extend(self.generate_buy_order(ctx, symbol)),
            }
        }

        AgentAction { orders }
    }

    fn on_fill(&mut self, trade: &Trade) {
        // Standard position tracking (same as TreeAgent)
        if trade.buyer_id == self.id {
            self.state.on_buy(trade.symbol.clone(), trade.quantity, trade.value());
        }
        if trade.seller_id == self.id {
            self.state.on_sell(trade.symbol.clone(), trade.quantity, trade.value());
        }
    }

    fn state(&self) -> &AgentState { &self.state }
}
```

### 3. MultiDiscrete Action Space (Per-Symbol Independent)

**Pattern**: N independent 3-way discrete actions (NOT 3^N joint combinations)

```rust
impl TradingEnv {
    /// Returns MultiDiscrete action space dimensions
    pub fn action_space(&self) -> Vec<usize> {
        vec![3; self.config.rl_symbols.len()]  // [3, 3, 3] for 3 symbols
    }

    /// Example for 3 symbols:
    /// - Action space: MultiDiscrete([3, 3, 3])
    /// - step([0, 1, 2]) → Sell AAPL, Hold GOOGL, Buy MSFT
    /// - NOT Discrete(27) with single joint action index
}
```

**Why MultiDiscrete:**
- Factored action space scales linearly (not exponentially) with N symbols
- Matches ML agent pattern: one policy, per-symbol features → per-symbol actions
- Standard for multi-asset RL (FinRL, TradingGym)
- Natural: "For each symbol, given its features, should I buy/sell/hold?"

impl RewardFunction for PnlDeltaReward {
    fn compute(&self, prev: &EpisodeView, _action: &dyn ActionSpace, curr: &EpisodeView) -> f64 {
        curr.equity() - prev.equity()
    }
}

**V7.1 implementation** (prepared, not used in V6):
```rust
pub struct RiskAdjustedReward {
    pub drawdown_penalty: f64,
    pub transaction_cost_per_trade: f64,
    pub sharpe_terminal_bonus: f64,
}

impl RewardFunction for RiskAdjustedReward {
    fn compute(&self, prev: &EpisodeView, action: &dyn ActionSpace, curr: &EpisodeView) -> f64 {
        let pnl = curr.equity() - prev.equity();
        let drawdown = (curr.peak_equity - curr.equity()).max(0.0) * self.drawdown_penalty;
        let tx_cost = if is_trade_action(action) { self.transaction_cost_per_trade } else { 0.0 };
        pnl - drawdown - tx_cost
        // Terminal Sharpe bonus added in is_terminal() check
    }
}
```

### 5. Episode Management (MODIFIED - Reference Pattern)

**Original problem**: EpisodeState duplicates AgentState data (cash, realized_pnl, etc.)

**New design**: Reference pattern with on-demand computation

```rust
/// Lightweight episode view - no data duplication
pub struct EpisodeView<'a> {
    tick: u64,
    agent_state: &'a AgentState,
    symbol: &'a Symbol,
    mid_price: Price,
    peak_equity: f64,  // Only gym-specific state
}

impl<'a> EpisodeView<'a> {
    pub fn equity(&self) -> f64 {
        let position_value = self.agent_state.position_for(self.symbol) as f64 * self.mid_price.to_float();
        self.agent_state.cash().to_float() + position_value
    }

    pub fn cash(&self) -> f64 { self.agent_state.cash().to_float() }
    pub fn realized_pnl(&self) -> f64 { self.agent_state.realized_pnl().to_float() }
    pub fn tick(&self) -> u64 { self.tick }
}

pub struct TradingEnvConfig {
    pub sim_config: SimulationConfig,
    pub agent_config: ExternalAgentConfig,
    pub rl_symbols: Vec<Symbol>,  // Which symbols RL agent observes/trades
    pub max_ticks: u64,           // Episode length (default: 10_000)
    pub seed: Option<u64>,        // None = random each reset
    pub reward_fn: Box<dyn RewardFunction>,
}
```

**Termination conditions** (unchanged):
- `tick_count >= max_ticks` → terminated (natural end)
- `equity <= 0` → terminated (bankrupt)

### Simulation Construction via Agent Factory

**Original problem**: Gym manually builds simulation + spawns agents, duplicating factory logic

**New design**: Delegate to agent_factory

```rust
pub struct TradingEnv {
    sim: Option<Simulation>,     // None before first reset
    agent_id: AgentId,
    extractor: Box<dyn FeatureExtractor>,  // CanonicalFeatures by default
    reward_fn: Box<dyn RewardFunction>,
    config: TradingEnvConfig,
    tick_count: u64,
    peak_equity: f64,
    prev_equity: f64,            // ADDED: Track previous equity for correct reward computation
    ml_models: MlModels,         // For agent_factory
}

impl TradingEnv {
    pub fn new(config: TradingEnvConfig) -> Self {
        let initial_cash = config.agent_config.initial_cash.to_float();

        Self {
            sim: None,
            agent_id: AgentId(u64::MAX - 1),  // EXTERNAL_AGENT_ID constant
            extractor: Box::new(CanonicalFeatures::new()),  // V6.3 default
            reward_fn: config.reward_fn,
            config,
            tick_count: 0,
            peak_equity: initial_cash,
            prev_equity: initial_cash,  // ADDED: Initialize to starting cash
            ml_models: MlModels::new(),  // Empty or load from config
        }
    }

    pub fn reset(&mut self, seed: Option<u64>) -> Vec<f64> {
        let actual_seed = seed.or(self.config.seed);

        // Create fresh simulation with seed
        let sim_config = if let Some(s) = actual_seed {
            let mut cfg = self.config.sim_config.clone();
            cfg.seed = s;
            cfg
        } else {
            self.config.sim_config.clone()
        };

        let mut sim = Simulation::new(sim_config.clone());

        // Create external agent
        let external_agent = ExternalAgent::new(
            self.agent_id,
            self.action_space.clone(),
            self.config.agent_config.clone(),
        );

        // Delegate to agent_factory (spawns background agents + adds external agent)
        let mut rng = if let Some(s) = actual_seed {
            rand::rngs::StdRng::seed_from_u64(s)
        } else {
            rand::rngs::StdRng::from_entropy()
        };

        agent_factory::spawn_with_external_agent(
            &mut sim,
            &sim_config,
            &self.ml_models,
            &mut rng,
            Box::new(external_agent),
        );

        // Warmup: let simulation determine its own needs
        let warmup_ticks = sim.warmup_ticks_needed();
        for _ in 0..warmup_ticks {
            sim.step();
        }

        self.sim = Some(sim);
        self.tick_count = 0;
        let initial_cash = self.config.agent_config.initial_cash.to_float();
        self.peak_equity = initial_cash;
        self.prev_equity = initial_cash;  // ADDED: Reset prev_equity

        self.extract_observation()
    }

    pub fn step(&mut self, action_indices: Vec<usize>) -> StepResult {
        // 1. Parse multi-symbol actions (MultiDiscrete: N independent 3-way actions)
        // action_indices.len() == N symbols, each action_indices[i] in [0, 1, 2]
        let actions: HashMap<Symbol, DiscreteAction> = self.config.rl_symbols.iter()
            .zip(action_indices.iter())
            .map(|(sym, &idx)| (sym.clone(), DiscreteAction::from_idx(idx)))
            .collect();

        // 2. Set actions in external agent (clean mutation before tick)
        let sim = self.sim.as_mut().expect("Simulation not initialized");
        if let Some(agent) = sim.get_external_agent_mut(self.agent_id) {
            agent.set_next_actions(actions);
        }

        // 3. Advance simulation one tick
        sim.step();
        self.tick_count += 1;

        // 4. Extract observation (concatenated multi-symbol features)
        let obs = self.extract_observation();

        // 5. Compute reward (portfolio-level P&L delta)
        let sim = self.sim.as_ref().unwrap();
        let agent_state = sim.agent_state(self.agent_id).expect("RL agent exists");

        // Compute current portfolio equity
        let position_values: f64 = self.config.rl_symbols.iter()
            .map(|symbol| {
                let position = agent_state.position_for(symbol);
                let mid_price = sim.market().mid_price(symbol).unwrap_or(Price::from_dollars(100));
                position as f64 * mid_price.to_float()
            })
            .sum();
        let curr_equity = agent_state.cash().to_float() + position_values;

        self.peak_equity = self.peak_equity.max(curr_equity);

        // FIXED: Use explicit prev_equity tracking (not approximation)
        let reward = curr_equity - self.prev_equity;
        self.prev_equity = curr_equity;

        // 6. Check termination
        let terminated = self.tick_count >= self.config.max_ticks || curr_equity <= 0.0;

        StepResult {
            observation: obs,
            reward,
            terminated,
            truncated: false,
            info: StepInfo {
                tick: self.tick_count,
                equity: curr_equity,
                realized_pnl: agent_state.realized_pnl().to_float(),
            },
        }
    }

    pub fn observation_size(&self) -> usize {
        self.extractor.n_features() * self.config.rl_symbols.len()
    }

    pub fn action_space(&self) -> Vec<usize> {
        vec![3; self.config.rl_symbols.len()]  // MultiDiscrete: [3, 3, 3] for 3 symbols
    }

    pub fn observation_space_bounds(&self) -> (Vec<f64>, Vec<f64>) {
        let registry = self.extractor.registry();
        let ranges = registry.valid_ranges();
        let (low, high): (Vec<_>, Vec<_>) = ranges.iter().cloned().unzip();
        (low, high)
    }
}
```

### Observation Extraction via ML Cache (MODIFIED)

**Original problem**: Plan shows `todo!()` with two options, doesn't commit

**Decision**: **Option A (ML cache) is the correct approach**

**Evidence from codebase exploration**:
- ML cache fully implemented in V5.6
- Features extracted in Phase 3 of tick loop
- Same extractor used for cache and recording (training-serving parity)
- CanonicalFeatures (28) is V6.3 default

```rust
impl TradingEnv {
    fn extract_observation(&self) -> Vec<f64> {
        let sim = self.sim.as_ref().expect("Simulation not initialized");

        // Multi-symbol: Concatenate features for each symbol
        self.config.rl_symbols.iter()
            .flat_map(|symbol| self.extract_symbol_features(sim, symbol))
            .collect()
    }

    fn extract_symbol_features(&self, sim: &Simulation, symbol: &Symbol) -> Vec<f64> {
        // Option A: Read from ML cache (features already extracted in Phase 3)
        if let Some(cache) = sim.ml_cache() {
            if let Some(features) = cache.get_features(symbol) {
                return features.to_vec();
            }
        }

        // Fallback: Manual extraction (during warmup or if cache not configured)
        // Uses same extractor as cache would use - training-serving parity maintained
        let ctx = self.build_temp_context(sim);
        let mut features = self.extractor.extract_market(symbol, &ctx);
        impute_features(&mut features, self.extractor.neutral_values());
        features.to_vec()
    }

    fn build_temp_context<'a>(&'a self, sim: &'a Simulation) -> StrategyContext<'a> {
        // Minimal context for fallback extraction (warmup period only)
        StrategyContext::new(
            sim.tick(),
            sim.timestamp(),
            sim.market(),
            sim.all_candles(),
            sim.indicator_engine().snapshot(),
            sim.all_recent_trades(),
            sim.active_events(),
            sim.fundamentals(),
        )
    }
}
```

**Why Option A is superior**:
- **No new simulation APIs** - Uses existing ML cache
- **Training-serving parity** - Same extractor, same features as recording
- **Efficient** - Features already computed in Phase 3
- **Open-closed** - New feature types extend FeatureExtractor, not simulation

| **Observation extraction** | `todo!()` with unclear decision | ML cache (Option A) with fallback | **Open-closed**: No sim modifications; **Parity**: Same extractor as training |
| **Episode state** | Copy AgentState fields to EpisodeState | Reference pattern with EpisodeView<'a> | **SoC**: State lives in simulation; **Efficient**: Zero allocations |
| **Action extensibility** | Hardcoded DiscreteAction enum | ActionSpace trait | **Open-closed**: V7 continuous extends, doesn't modify |
| **Warmup logic** | Hardcoded 100 ticks in gym | `Simulation::warmup_ticks_needed()` | **Declarative**: Sim knows needs; **SoC**: Warmup is sim concern |

## Part 2: PyO3 Bindings (Python ↔ Rust)

See `6.5_pyo3_bindings.md` for detailed code examples. This section summarizes key design decisions and implementation strategy.

### Design Decisions

#### 1. GIL Release During Rust Computation

All Rust computation runs inside `py.allow_threads()`, releasing the Python GIL. This is critical for:
- **`PyVecTradingEnv`**: Multiple environments step in parallel via rayon
- **Python-side threading**: Training code can do I/O while Rust computes
- **Performance**: Eliminates GIL contention for parallel episode collection

#### 2. NumPy Array Interface

Feature vectors returned as `numpy.ndarray` (via the `numpy` crate). Zero-copy where possible.
- `reset()` returns `np.ndarray` shape `(n_features,)`, dtype `float64`
- `step()` returns `(np.ndarray, float, bool, bool, dict)` matching gymnasium

#### 3. Gymnasium Compatibility

`observation_space` and `action_space` properties return actual `gymnasium.spaces` objects:
- **observation_space**: `gymnasium.spaces.Box(low=-inf, high=inf, shape=(n_features,), dtype=float64)`
- **action_space**: `gymnasium.spaces.MultiDiscrete([3, 3, 3])` for N=3 symbols

This enables direct use with `stable-baselines3` without wrapper classes.

#### 4. Vectorized Environment

`PyVecTradingEnv` runs N independent simulations in parallel using rayon. Essential for V7.2 RL training where throughput depends on environment parallelism.

### Core Types

#### PyTradingEnv (Single Environment)

Wraps `gym::TradingEnv` with Python interface:
- `__init__(max_ticks, seed, rl_symbols)`: Create environment
- `reset(seed)`: Returns `np.ndarray` observation
- `step(actions)`: Takes list of action indices (one per symbol), returns `(obs, reward, terminated, truncated, info)`
- `observation_space`, `action_space`: Properties returning gymnasium.spaces objects
- `feature_names()`: Returns list of feature names for debugging

#### PyVecTradingEnv (Parallel Environments)

Runs N environments in parallel via rayon:
- `__init__(num_envs, max_ticks, base_seed)`: Create N parallel environments
- `reset_all(seeds)`: Returns 2D array `[num_envs, n_features]`
- `step_all(actions)`: Takes 2D actions `[num_envs, n_symbols]`, returns batched results
- Fully parallel: All environments step simultaneously using rayon
